# v1.2.0 — 穩定性修復與環境優化 (Stability & Environment Patch)

Helix Bio v1.2.0 是一個關鍵的穩定性更新版本，解決了 v1.1.0 在佈署與環境整合時遇到的核心問題，並確保了所有新功能在本地開發環境中的完美執行。

## 關鍵修復與優化

- **資料庫連線穩定性 (DB Authorization Fix)**
  - 修正了 `.env` 中資料庫角色不符的問題，將預設角色從 `postgres` 調整為本地開發者角色 `tuchengshin`，解決了 500 Internal Server Error。

- **服務埠口遷移 (Port Migration)**
  - **後端 (FastAPI)**：遷移至 **8001** 埠口，避開 macOS 預設 8000 埠口的衝突。
  - **前端 (Vite)**：遷移至 **5555** 埠口，提供更乾淨的開發環境，並同步更新了代理 (Proxy) 配置。

- **核心程式碼補全 (API Router Restoration)**
  - 修復了 `backend/api/genes.py` 中遺失的 `router` 定義，恢復了基因搜尋功能的運作。

- **功能落地補強 (Consistency Reinforcement)**
  - 確保了「每日 100 次查詢限制」與「搜尋歷史記錄」在變體與路徑端點的完整執行與驗證。

## 本版本包含

- v1.1.0 的所有安全性與一致性增強。
- 針對 500 錯誤與環境衝突的全面修復。
- 經過驗證的跨端點查詢配額控管。
- 清理後的專案環境（移除了 `test_regulome.py`）。

## 適合誰使用

強烈建議所有使用者升級至 v1.2.0。此版本解決了 v1.1.0 中的啟動與連線問題，是目前最穩定且功能最完備的版本。

<task type="auto">
  <name>Refactor OAuth Callback to persist User</name>
  <files>backend/api/auth.py</files>
  <action>
    - Import `User` model and `get_db` session dependency.
    - In `callback` endpoint for both Google and GitHub:
      1. Check if a user with the same `provider` and `provider_id` exists.
      2. If not, create a new `User` record with email, name, and avatar_url.
      3. If yes, update existing fields (email, name, avatar_url) if changed.
      4. Commit changes to the database.
      5. Create the JWT `access_token` with `sub` set to `str(user.id)` (the internal UUID) instead of the provider's ID.
    - Ensure all DB operations are asynchronous.
  </action>
  <verify>
    <automated>pytest backend/tests/test_auth.py</automated>
  </verify>
  <done>User record created/updated in DB and JWT sub matches internal User.id.</done>
</task>

<task type="auto">
  <name>Refactor get_current_user dependency</name>
  <files>backend/auth/dependencies.py</files>
  <action>
    - Import `User` model, `AsyncSession`, and `get_db`.
    - Update `get_current_user` and `get_optional_user` signatures to accept `db: AsyncSession = Depends(get_db)`.
    - After verifying the token payload, extract `sub` (the user UUID).
    - Query the database: `select(User).where(User.id == uuid.UUID(payload["sub"]))`.
    - Return the `User` object (or raise 401 if not found for `get_current_user`).
    - Note: This ensures downstream endpoints have access to the full user object (per D-03).
  </action>
  <verify>
    <automated>pytest backend/tests/test_auth.py</automated>
  </verify>
  <done>get_current_user returns a User model instance.</done>

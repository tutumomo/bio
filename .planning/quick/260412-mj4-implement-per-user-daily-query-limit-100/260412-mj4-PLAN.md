# Quick Task 260412-mj4: Implement per-user daily query limit

**Description:** Implement per-user daily query limit (100/day) in `backend/api/genes.py` as required by SETUP-04.

## Plan

### Task 1: Update `backend/api/genes.py` to enforce daily limit
- Import `select` from `sqlalchemy`.
- Import `HTTPException` from `fastapi`.
- Import `User` from `backend.models.user`.
- Import `date` from `datetime`.
- Update `search_genes` endpoint:
    - If `current_user` is present:
        - Fetch `User` object from DB.
        - Check `last_query_date`. If not today, reset `daily_query_count` to 0 and update `last_query_date` to today.
        - Check `daily_query_count`. If >= 100, raise `HTTPException(status_code=429, detail="Daily query limit reached (100/day)")`.
        - Increment `daily_query_count`.
        - Save `SearchHistory` entry as before.
        - Commit changes.

### Task 2: Create a verification test
- Create `backend/tests/test_genes_api.py` to verify the daily limit enforcement.
- Test scenarios:
    - Unauthenticated search (no limit).
    - Authenticated search within limit.
    - Authenticated search exceeding limit (should return 429).
    - Daily reset logic (searching on a new day resets the count).

## Verification Strategy
- Run `pytest backend/tests/test_genes_api.py`.
- Verify the search endpoint returns 429 after 100 queries for the same user.
